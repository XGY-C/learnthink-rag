# 人工智能导论 — 第3章 贝叶斯分类器

## 3.1 贝叶斯理论基础

### 3.1.1 频率学派与贝叶斯学派

在概率论与统计学中，对概率的解释存在两个主要的学派：
- **频率学派**：认为概率是客观的，是事件在大量重复试验中出现的频率极限。他们只依赖观测数据，认为模型参数是未知的确定值。
- **贝叶斯学派**：认为概率是主观的，是对事件发生置信度的度量。他们引入了**先验知识**，并认为模型参数是随机变量，通过观测数据（证据）来不断更新对参数的置信度（后验）。

贝叶斯分类器正是基于贝叶斯学派的思想：**结合先验知识与观测数据，得到更可靠的后验推断**。

### 3.1.2 贝叶斯定理

**贝叶斯定理**（Bayes' Theorem）是概率论中的基本定理，描述了在已知某些条件下某事件发生的概率。

**推导**：由条件概率的定义可知联合概率可表示为：
- $P(A, B) = P(A|B) \times P(B)$
- $P(A, B) = P(B|A) \times P(A)$

两式相等，整理即可得到贝叶斯定理公式：

**公式：$P(A|B) = \frac{P(B|A) \times P(A)}{P(B)}$**

其中：
- **$P(A|B)$**：后验概率，给定观测证据 B 发生时，假设 A 成立的概率
- **$P(B|A)$**：似然度，在假设 A 成立的条件下，观测到证据 B 的概率
- **$P(A)$**：先验概率，在没有任何观测证据前，对假设 A 成立的置信度
- **$P(B)$**：证据因子/边缘概率，在任何假设下观测到证据 B 的总概率

### 3.1.3 先验与后验的直观案例

已知某种疾病在人群中的患病率为 1%（先验），检测准确率为 99%（即患病者99%呈阳性，健康者99%呈阴性）。如果某人检测为阳性，他实际患病的概率是多少？

设患病为 $D$，阳性为 $T$。根据贝叶斯定理：
$$P(D|T) = \frac{P(T|D) \times P(D)}{P(T|D)P(D) + P(T|\neg D)P(\neg D)}$$

代入数据：
$$P(D|T) = \frac{0.99 \times 0.01}{0.99 \times 0.01 + 0.01 \times 0.99} = \frac{0.0099}{0.0099 + 0.0099} = 50\%$$

**解释**：尽管检测准确率高达 99%，但呈阳性时真正患病的概率只有 50%！这是因为健康人基数太大（99%），导致即使只有 1% 的误诊率，假阳性的人数（0.99万人）也和真阳性人数（0.99万人）一样多。这就是先验概率对后验概率的强大修正作用。

## 3.2 朴素贝叶斯分类器

### 3.2.1 基本思想与MAP准则

**朴素贝叶斯分类器**（Naive Bayes Classifier）基于贝叶斯定理，假设各特征之间**条件独立**。

对于给定样本 $x = (x_1, x_2, ..., x_n)$，属于类别 $C_k$ 的后验概率为：
$$P(C_k | x) = \frac{P(C_k) \times P(x | C_k)}{P(x)}$$

由于特征条件独立假设，似然度可展开为连乘：
$$P(C_k | x_1,...,x_n) = \frac{P(C_k) \times \prod_{i=1}^n P(x_i | C_k)}{P(x)}$$

**分类决策准则（最大后验概率 MAP）**：由于分母 $P(x)$ 对所有类别都是相同的，我们在分类时只需比较分子部分，选择后验概率最大的类别：
$$h_{nb}(x) = \arg\max_{C_k} \left( P(C_k) \times \prod_{i=1}^n P(x_i | C_k) \right)$$

### 3.2.2 "朴素"假设的含义与现实意义

"**朴素**"（Naive）指的是：假设所有特征在给定类别的条件下互相独立。

**现实比喻**：假设要判断一个人是否是程序员（类别），看“戴眼镜”和“穿格子衫”（特征）。实际上这两个特征在程序员群体中并不独立（经常一起出现），但它们各自都能提供正向的分类信号。朴素贝叶斯忽略了特征间的相关性，虽然概率估计值偏差较大，但只要**各类别概率的排序正确**，分类结果就是正确的。

在实践中朴素贝叶斯效果很好的原因：
1. 分类决策只依赖于后验概率的排序，而非精确值
2. 对小样本数据有较好的泛化能力，参数少，不容易过拟合

### 3.2.3 三种常见变体

**1. 高斯朴素贝叶斯（GaussianNB）**：
- 假设特征服从**正态分布**：$P(x_i | C_k) = \frac{1}{\sqrt{2\pi\sigma_{k,i}^2}} \exp\left(-\frac{(x_i - \mu_{k,i})^2}{2\sigma_{k,i}^2}\right)$
- 适用场景：特征为连续值，如身高、温度
- 参数估计：计算每个类别下各特征的均值 $\mu$ 和方差 $\sigma^2$

**2. 多项式朴素贝叶斯（MultinomialNB）**：
- 特征表示计数（如词频）
- 适用场景：**文本分类**、文档主题识别
- 典型应用：垃圾邮件过滤

**3. 伯努利朴素贝叶斯（BernoulliNB）**：
- 特征为二元值（0/1，出现/不出现）
- 适用场景：短文本分类、情感分析

### 3.2.4 拉普拉斯平滑

**零概率问题**：如果某个特征值在训练集中的某个类别下未出现，其条件概率会被估计为 0，由于是连乘，这将导致整个后验概率为 0，抹杀其他所有特征的信息。

**拉普拉斯平滑**（Laplace Smoothing）通过在计数中加入一个正常数来避免零概率：

公式：$P(x_i | C_k) = \frac{count(x_i, C_k) + \alpha}{count(C_k) + \alpha \times V}$

- $\alpha$：平滑参数（$\alpha=1$ 为标准拉普拉斯平滑，$\alpha<1$ 为利德斯通平滑）
- $V$：特征可能的取值数量（词汇表大小）

### 3.2.5 工程避坑：下溢问题与对数变换

在实际计算中，特征维度往往成千上万，多个 $P(x_i|C_k)$ 连乘的结果极易接近于 0，导致计算机浮点数**下溢**，被系统直接当作 0 处理。

**解决方案**：利用对数将连乘转化为连加。因为 $\log$ 是单调递增函数，取对数后不会改变概率大小的排序，因此不影响分类结果：
$$\log P(C_k | x) \propto \log P(C_k) + \sum_{i=1}^n \log P(x_i | C_k)$$

## 3.3 半朴素贝叶斯分类器

**半朴素贝叶斯分类器**（Semi-Naive Bayes）放松了完全条件独立的假设，允许部分特征之间存在依赖关系，是"全独立"与"全依赖"的折中。

常见策略：
- **SPODE**（Super-Parent ODE）：所有特征依赖同一个"超父"特征
- **TAN**（Tree-Augmented Naive Bayes）：特征间形成树形依赖结构（基于最大带权生成树算法构建）
- **AODE**（Averaged ODE）：集成多个 SPODE 的结果，通过平均避免依赖单一超父特征

## 3.4 贝叶斯网络

**贝叶斯网络**（Bayesian Network）是一种概率图模型，用**有向无环图**（DAG）表示变量之间的条件依赖关系。

### 3.4.1 结构组成
- **节点**：表示随机变量
- **有向边**：表示变量之间的依赖关系（因果或影响）
- **条件概率表**（CPT）：每个节点在给定父节点取值条件下的概率分布

### 3.4.2 条件独立性判断（D-分离）
贝叶斯网络通过图结构编码了变量间的独立性，判断变量是否条件独立的核心法则是 **D-分离**。常见的三种连接结构：
1. **顺连（A → B → C）**：给定 B，A 和 C 条件独立
2. **分连（A ← B → C）**：给定 B，A 和 C 条件独立
3. **汇连（A → B ← C）**：未给定 B（及其后代）时，A 和 C 独立；给定 B 时，A 和 C 反而条件依赖（解释推理的来源）

### 3.4.3 推理类型
- **因果推理**（Causal）：从原因推断结果（顺流方向）
- **诊断推理**（Diagnostic）：从结果推断原因（逆流方向）
- **解释推理**（Intercausal）：多个原因之间的交互（汇连结构中的竞争解释）

## 3.5 实战：朴素贝叶斯文本分类

### 3.5.1 文本分类流程

1. **文本预处理**：分词、去停用词
2. **特征构建**：词袋模型或 TF-IDF
3. **训练**：计算每个类别的先验概率 $P(C_k)$ 和条件概率 $P(w|C_k)$
4. **预测**：对每条新文本，计算所有类别的后验概率（实际取对数连加），选择最大值

### 3.5.2 代码示例（Python）

以下代码展示了完整的中文文本分类流程，包含中文分词处理与概率输出：

```python
from sklearn.naive_bayes import MultinomialNB
from sklearn.feature_extraction.text import CountVectorizer
import jieba

# 1. 准备数据
texts = ["人工智能改变世界", "机器学习是人工智能的核心",
         "今天天气很好", "周末去哪里玩"]
labels = ["科技", "科技", "生活", "生活"]

# 2. 中文分词预处理（关键！CountVectorizer默认按空格切分，对中文无效）
texts_cut = [" ".join(jieba.lcut(text)) for text in texts]

# 3. 向量化（词袋模型）
vectorizer = CountVectorizer()
X = vectorizer.fit_transform(texts_cut)

# 4. 训练模型
# alpha=1.0 启用拉普拉斯平滑；sklearn内部已自动处理对数变换，无需手动取对数
clf = MultinomialNB(alpha=1.0) 
clf.fit(X, labels)

# 5. 预测新文本
new_text = ["深度学习的发展趋势"]
new_text_cut = [" ".join(jieba.lcut(text)) for text in new_text]
X_new = vectorizer.transform(new_text_cut)

# 输出预测类别
print("预测类别:", clf.predict(X_new))  # ['科技']

# 输出后验概率的对数值（由于是连乘计算，sklearn输出的是取对数后的概率）
print("对数后验概率:", clf.predict_log_proba(X_new))

# 输出映射为0-1的概率值
print("后验概率:", clf.predict_proba(X_new)) 
# [[0.24... 0.75...]] -> 属于"生活"的概率约24%，"科技"的概率约75%
