# 人工智能导论 — 第2章 搜索算法

## 2.1 搜索问题概述

**搜索**是人工智能中最基本的推理技术之一。许多 AI 问题（如路径规划、拼图求解、博弈对弈）都可以形式化为搜索问题：从初始状态出发，通过一系列动作，找到到达目标状态的路径。

### 2.1.1 搜索问题的形式化

一个搜索问题由以下要素定义：
- **状态空间**（State Space）：所有可能状态的集合。
- **初始状态**（Initial State）：搜索的起点。
- **目标测试**（Goal Test）：判断当前状态是否为目标。
- **后继函数**（Successor Function）：返回从某个状态出发可到达的状态集合及对应的动作。
- **路径代价**（Path Cost）：每条路径的代价，通常用函数 $g(n)$ 表示从起点到节点 $n$ 的实际代价。

> **💡 概念辨析**：BFS 的最优性前提是“每一步的代价相同”（即步数代价）。如果不同动作的代价不同（例如平地移动代价为1，上山移动代价为3），BFS 无法保证找到最优解，此时必须使用一致代价搜索（UCS）。

### 2.1.2 搜索策略分类

- **盲目搜索**（Uninformed Search）：不使用问题特定知识，仅按预定策略展开。
- **启发式搜索**（Informed Search）：使用启发函数 $h(n)$ 估计到目标的距离，引导搜索方向。

---

## 2.2 盲目搜索算法

### 2.2.1 广度优先搜索 (BFS)

**广度优先搜索**按层次逐层展开搜索树，先访问离根节点最近的节点。使用**队列**作为边缘的数据结构。

**遍历示例**：
```text
    1
   / \
  2   3
 / \   \
4   5   6

BFS 遍历顺序: 1 -> 2 -> 3 -> 4 -> 5 -> 6
```

**特点**：
- **完备性**：是（如果解存在，BFS 一定能找到）
- **最优性**：是（仅当每步代价相同时保证最优）
- **时间复杂度**：$O(b^d)$，其中 $b$ 为分支因子，$d$ 为解的深度
- **空间复杂度**：$O(b^d)$，需要存储整层节点，内存消耗极大

### 2.2.2 深度优先搜索 (DFS)

**深度优先搜索**优先探索最深的分支，到达叶节点后回溯。使用**栈**作为边缘的数据结构。

**遍历示例**（同上图）：
```text
DFS 遍历顺序: 1 -> 2 -> 4 -> 5 -> 3 -> 6
```

**特点**：
- **完备性**：否（在无限深度空间中可能陷入死循环）
- **最优性**：否（找到的不一定是最短路径）
- **时间复杂度**：$O(b^m)$，$m$ 为最大深度
- **空间复杂度**：$O(bm)$，只需要存储当前路径，空间效率高

### 2.2.3 迭代加深搜索 (IDS)

**迭代加深搜索**结合了 DFS 的空间优势和 BFS 的完备性。每次执行有深度限制的 DFS，逐轮增大深度上限（0, 1, 2, ...）。

> **💡 消除悖论**：初学者常认为 IDS 重复搜索上层节点会导致效率低下。但由于搜索树中节点数量随深度呈指数增长，底层节点数远大于上层节点数，因此重复搜索的时间开销其实很小，其时间复杂度仍为 $O(b^d)$，但空间复杂度降到了 $O(bd)$。

### 2.2.4 一致代价搜索 (UCS)

**一致代价搜索**按照从起点累积的路径代价 $g(n)$ 来扩展节点，是 Dijkstra 算法在搜索问题中的应用。使用**优先队列（最小堆）**实现。

**特点**：
- **完备且最优**（前提：边的代价严格大于0）
- 扩展节点时不仅看深度，更看重已消耗的实际代价。

## 2.3 启发式搜索算法

### 2.3.1 贪心最佳优先搜索

**贪心最佳优先搜索**使用启发函数 $h(n)$ 估计从节点 $n$ 到目标的距离，每次扩展 $h(n)$ 最小的节点。

**特点**：
- 速度快，但只顾眼前利益，可能不是最优解，容易陷入局部最优（如死胡同）。

### 2.3.2 A* 搜索算法

**A\* 搜索**是最著名的启发式搜索算法，使用评价函数 **$f(n) = g(n) + h(n)$**：
- **$g(n)$**：从起点到 $n$ 的实际代价（向后看，已付出的）
- **$h(n)$**：从 $n$ 到目标的估计代价（向前看，启发式的）

A* 算法优先扩展 $f(n)$ 最小的节点，既考虑了已付出的代价，又考虑了未来的预期。

#### A* 的最优性条件

A* 保证找到最优解的条件取决于 $h(n)$ 的性质：
1. **可采纳性**（Admissibility）：$h(n)$ 永远不高估实际代价，即 $h(n) \le h^*(n)$。这保证了 A* 在树搜索中是最优的。
2. **一致性**（Consistency/Monotonicity）：$h(n) \le c(n, n') + h(n')$（即满足三角不等式）。这保证了 A* 在图搜索（有闭集检查）中是最优的，且不需要重复扩展节点。**一致性是比可采纳性更强的条件。**

#### 常见启发函数
- **曼哈顿距离**：$|x_1-x_2| + |y_1-y_2|$，适用于只能上下左右移动的网格地图。
- **欧几里得距离**：$\sqrt{(x_1-x_2)^2 + (y_1-y_2)^2}$，适用于可任意方向移动的地图。
- **错位方块数**：8 数码问题中不在目标位置的方块数量（常需除以2以满足一致性）。

#### A* 算法 Python 实现

```python
import heapq

def a_star_search(graph, start, goal, h_func):
    """
    graph: 图的邻接表表示，格式为 {节点: {邻居: 边代价}}
    start: 起始节点
    goal: 目标节点
    h_func: 启发函数，输入节点返回到目标的估计代价
    """
    # 优先队列，元素格式为 (f_value, g_value, current_node, path)
    open_list = [(h_func(start), 0, start, [start])]
    closed_list = set()

    while open_list:
        f, g, current, path = heapq.heappop(open_list)

        # 目标测试
        if current == goal:
            return path, g  # 返回路径和总代价

        # 图搜索：跳过已扩展节点
        if current in closed_list:
            continue
        closed_list.add(current)

        # 扩展邻居
        for neighbor, cost in graph.get(current, {}).items():
            if neighbor not in closed_list:
                new_g = g + cost
                new_f = new_g + h_func(neighbor)
                heapq.heappush(open_list, (new_f, new_g, neighbor, path + [neighbor]))

    return None, float('inf')  # 未找到路径
```

### 2.3.3 IDA* 算法

**IDA\***（Iterative Deepening A*）用 $f$ 值作为阈值进行迭代加深搜索。每一轮将阈值设为上一轮超过旧阈值的最小 $f$ 值。兼具 A* 的启发能力和 DFS 的空间效率 $O(bd)$。

## 2.4 对抗搜索

对抗搜索用于存在竞争关系的多智能体环境（如棋类游戏）。

### 2.4.1 Minimax 算法

**Minimax** 用于两人零和博弈。MAX 方希望最大化得分，MIN 方希望最小化得分，双方交替行动。算法递归地为每个节点计算极小极大值。

**博弈树示例**：
```text
层级:    MAX         A
         /         \
MIN    B(3)        C(2)
       / \         / \
MAX  D    E      F    G
     |    |      |    |
终端 3   12      2    8

解释：D返回3，E返回12；B是MIN节点，选最小值3；F返回2，G返回8；C是MIN节点，选最小值2；A是MAX节点，选最大值3。
```

### 2.4.2 Alpha-Beta 剪枝

**Alpha-Beta 剪枝**是 Minimax 的优化版本，通过维护两个阈值来剪除不可能被选择的子树：
- **$\alpha$ (Alpha)**：MAX 节点目前能保证的最佳值（下界），初始化为 $-\infty$
- **$\beta$ (Beta)**：MIN 节点目前能保证的最佳值（上界），初始化为 $+\infty$

**剪枝规则**：
1. 在 MIN 节点处，如果当前值 $\le \alpha$，则剪掉该节点其余分支（因为 MAX 父节点绝不会选这条路径）。
2. 在 MAX 节点处，如果当前值 $\ge \beta$，则剪掉该节点其余分支（因为 MIN 父节点绝不会选这条路径）。

**剪枝效果**：
- 最优情况下（节点排列完美），时间复杂度从 $O(b^d)$ 降到 $O(b^{d/2})$，即在相同时间内搜索深度翻倍。
- 实际效果严重依赖于子节点的排列顺序，常配合启发式排序使用。

## 2.5 算法对比与本章小结

### 核心算法对比表

| 算法 | 完备性 | 最优性 | 时间复杂度 | 空间复杂度 | 适用场景 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **BFS** | 是 | 是(步数代价) | $O(b^d)$ | $O(b^d)$ | 步数代价一致的最短路径问题 |
| **DFS** | 否 | 否 | $O(b^m)$ | $O(bm)$ | 内存受限，只需任一解的情况 |
| **IDS** | 是 | 是(步数代价) | $O(b^d)$ | $O(bd)$ | 内存受限，需最短路径的盲目搜索 |
| **UCS** | 是 | 是(非负代价) | $O(b^{C^*/\epsilon})$ | $O(b^{C^*/\epsilon})$ | 边代价不同的最短路径问题 |
| **A*** | 是 | 是(可采纳) | 取决于 $h(n)$ | 取决于 $h(n)$ | 有启发信息的路径规划、拼图求解 |

*(注：$C^*$为最优解代价，$\epsilon$为最小边代价，$m$为最大深度)*

### 扩展阅读：深度学习时代的搜索
传统的搜索算法在状态空间极大时面临组合爆炸。现代 AI 引入了深度学习来增强搜索：
- **蒙特卡洛树搜索 (MCTS)**：通过随机模拟评估节点价值，平衡探索与利用，是 AlphaGo 的核心算法。
- **学习型启发函数**：使用神经网络预测 $h(n)$ 或直接输出策略，如 DeepCubeA 在无人类知识的情况下独自学会还原魔方。

搜索算法按是否使用启发信息分为盲目搜索和启发式搜索。BFS 和 DFS 是基础的盲目搜索算法，A* 是最重要的启发式搜索算法，对抗搜索则是博弈论的基础。掌握这些算法是理解现代人工智能决策机制的基石。
